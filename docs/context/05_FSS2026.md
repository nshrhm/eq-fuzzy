# FSS 2026 context

## Role

FSS 2026 is the **membership-function specification and sensitivity-reporting**
workstream inside the private `eq-fuzzy` monorepo.

The short-note direction is:

> Membership Specification and Sensitivity Reporting for
> Fuzzy-Entropy-Based LLM Emotion Scoring

## Core question

When fuzzy entropy is computed from 0-100 LLM emotion scores, what
membership-function information must be specified, versioned, and
sensitivity-checked so that the entropy descriptor is reproducible and
interpretable?

## Scope

Use:

- score-axis membership grids
- entropy-response reporting units `H*(x)`
- membership specification cards
- minimal family-pair sensitivity reporting
- existing ICECCME, SCIS, or ICICIC artifacts only as motivation or aggregated
  provenance, not as reused main results

Do not use FSS 2026 to claim:

- a new model-ranking experiment
- a persona-temperature factorial result
- a human-grounded multilingual benchmark result
- an external benchmark-positioning result
- a target-shift result

## Required separation from SCIS

FSS 2026 does not repeat SCIS persona x temperature factorial deconfounding.
It must not reuse SCIS decomposition tables, interaction heatmaps,
model-level interaction burden, or Table IV-style cell-mean entropy sensitivity
as the main FSS result.

## Required separation from ICECCME

FSS 2026 does not repeat ICECCME human-grounded multilingual ranking,
Japanese-human VAS alignment, EN/ZH drift analysis, or the two-axis screening
view.

## Required separation from ICICIC

FSS 2026 does not repeat ICICIC benchmark positioning, benchmark coverage
matrices, external mini-comparisons, reader-side versus character-side target
shift, or target-shift tables.

## Main artifact

The main FSS artifact is a membership specification card plus an
entropy-response sensitivity report over the score axis.

Implementation anchors:

- `configs/fss/membership_specification_v1.yaml`
- `scripts/fss2026/`
- `artifacts/fss2026/membership_specification_v1/`
- `paper/fss2026/`
- `docs/fss2026/`

Do not call APIs, create raw JSONL files, copy SCIS raw outputs, or modify
submitted manuscript PDFs.
